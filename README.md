# SubSphere

**SubSphere** is an end-to-end multi-tenant SaaS Subscription Management platform built as a Hackathon prototype. It implements robust product variants, subscriptions state machines, quotation templates, invoicing automation, and active Razorpay integrations.

The entire web application stack is fully containerized and orchestrated seamlessly via Docker Compose.

---

## 🛠 Tech Stack

- **Backend / API**: Django 5, Django Rest Framework (DRF)
- **Frontend / UI**: Next.js 14 (App Router) with TypeScript
- **Database**: PostgreSQL 16
- **Reverse Proxy**: Nginx (Unifies architecture onto a single host port)
- **Payments**: Razorpay (Webhooks & Signature verification hooks)

---

## 🚀 Getting Started

The application is bundled natively with Docker, avoiding any requirement to run environments locally other than Docker. 

### 1. Start the Containers
Open a terminal in the root directory and initialize the infrastructure:
```bash
docker-compose up -d
```
> This command seamlessly spins up four containers (`frontend`, `backend`, `db`, and `nginx`) and creates a permanent data volume to persist your transactions securely.

### 2. View the Platform
Nginx runs natively as a reverse proxy, joining both the backend API and NextJS frontend into one unified link over port **8080**.

Navigate your browser here:
- **Platform Root**: [http://localhost:8080](http://localhost:8080)

*Note: Your `.env` variables dictate the Razorpay configurations and default application mapping.*

## 🗺️ Application Map

### Frontend Pages
The frontend is built using minimal styling strictly to demonstrate backend capabilities reliably.
- `/` — Index route (automatically redirects to the Dashboard page).
- `/login` — Login securely utilizing JSON Web Tokens (interfaced via cookies). 
- `/register` — Create a new Portal User via the Django `CustomUserManager`.
- `/dashboard` — Post-authentication central dashboard validating server-to-server data fetch. Displays loaded basic subscription data and metrics leveraging React effects.

### Backend Endpoints
Rather than reading raw Django views, the simplest way to interact with and test all 40+ system endpoints is via the autogenerated **Swagger UI Dashboard**.

- **Swagger Documentation**: [http://localhost:8080/api/docs/](http://localhost:8080/api/docs/)
  
> Swagger provides real-time POST capabilities allowing you to interact with API categories mapping out all core entities:
> - **Products & Plans** (Includes hierarchical variant pricing parameters)
> - **Subscriptions** (`/api/subscriptions/{id}/confirm` auto-spawns relational invoices!)
> - **Invoices & Payments** (HTML Hooks, verification payload simulators, and Taxes processing)
> - **Quotation Templates**
